- Memahami **peran penting proses review sebelum publikasi**
- Melatih praktik **peer review** terhadap notebook dan dokumentasi rekan
- Memberikan umpan balik yang **konstruktif, spesifik, dan dapat ditindaklanjuti**
- Menyiapkan notebook dan dokumentasi agar **siap diintegrasikan pada sesi berikutnya**

## Mengapa Review Itu Penting?

> **â€œAnalisis yang tidak direview berpotensi disalahpahami, meskipun kodenya benar.â€**

Dalam praktik profesional:

- Data scientist **hampir selalu bekerja dalam tim**
- Analisis harus **dapat dipahami oleh orang lain**, bukan hanya oleh penulisnya
- Kesalahan kecil seperti:
  - grafik tanpa judul,
  - asumsi yang tidak dijelaskan,
  - atau typo di README  
  dapat **menyesatkan interpretasi** dan menurunkan kredibilitas hasil analisis

Review membantu memastikan bahwa:
- insight tersampaikan dengan jelas,
- asumsi tersurat dengan baik,
- dan hasil analisis siap dibaca publik.

## Apa yang Direview pada Sesi Ini?

Karena sesi **Publikasi & Integrasi** belum dimulai, review difokuskan pada **versi draft** yang telah dikerjakan sebelumnya.

### 1. Kaggle Notebook (Draft)

Notebook yang direview diharapkan sudah memiliki:
- Exploratory Data Analysis (EDA) dasar:
  - histogram
  - scatter plot
  - heatmap korelasi
- Narasi storytelling dalam bentuk Markdown
- Status **diubah ke *public*** agar dapat diakses oleh reviewer

### 2. Repository GitHub (Draft)

Repository GitHub diharapkan memiliki:
- Struktur dasar proyek, minimal:
  - `data/`
  - `notebooks/`
  - `README.md`
- Isi `README.md` mencakup:
  - konteks proyek
  - deskripsi dataset
  - insight awal (boleh masih singkat)
- **Belum wajib** mencantumkan link Kaggle Notebook  
  (akan dilakukan pada sesi berikutnya)

## Alur Review Berpasangan

### Langkah 1: Menyiapkan Notebook untuk Direview

1. Buka notebook Anda di Kaggle
2. Klik tombol **Share** di kanan atas
3. Ubah akses menjadi **Public**
4. Salin URL notebook publik  
   (contoh: `https://www.kaggle.com/username/nama-notebook`)
5. Simpan perubahan

> ğŸ”’ **Catatan Privasi**  
> Setelah sesi selesai, Anda dapat mengembalikan notebook ke status *private*.  
> Namun, untuk keperluan workshop ini, **publikasi sementara sangat dianjurkan**.

---

### Langkah 2: Tukar Link dengan Pasangan

Setiap peserta menukar:
- Link **repository GitHub**
- Link **notebook Kaggle publik**

Pastikan kedua link dapat diakses tanpa izin khusus.

---

### Langkah 3: Review Kaggle Notebook

Gunakan daftar periksa berikut:

- [ ] Setiap visualisasi memiliki **penjelasan dalam Markdown**
- [ ] Alur cerita jelas: **konteks â†’ temuan â†’ kesimpulan**
- [ ] Bahasa mudah dipahami oleh **pembaca non-teknis**
- [ ] Tidak ada:
  - typo
  - grafik tanpa judul
  - kode error atau output membingungkan

---

### Langkah 4: Review Repository GitHub

Periksa hal-hal berikut:

- [ ] Struktur folder konsisten dan rapi
- [ ] `README.md` menjawab dengan jelas:
  - **Apa** tujuan proyek ini?
  - **Dataset** apa yang digunakan?
- [ ] Penjelasan cukup untuk orang lain memahami proyek tanpa membuka notebook terlebih dahulu

2. **Saran perbaikan yang spesifik**

3. **Pertanyaan klarifikasi (jika ada)**  
   Bagian mana yang masih ambigu atau perlu dijelaskan lebih lanjut?
