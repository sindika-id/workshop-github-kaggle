# Software License Reference

## Tujuan Sesi
- Memahami jenis-jenis lisensi perangkat lunak yang umum digunakan
- Mengetahui hak dan kewajiban setiap lisensi
- Mampu memilih lisensi yang sesuai untuk proyek
- Memahami lisensi dataset pada platform Kaggle

---

Dokumen ini berisi ringkasan jenis-jenis lisensi perangkat lunak yang umum digunakan,
beserta karakteristik, hak, dan batasannya.

Dokumen ini bersifat informatif dan tidak menggantikan teks lisensi resmi.
Untuk penggunaan hukum, selalu rujuk ke dokumen lisensi asli.

---

## 1. MIT License

**Jenis:** Permissive License

**Deskripsi:**
MIT License mengizinkan penggunaan, modifikasi, distribusi, dan komersialisasi
perangkat lunak tanpa kewajiban membuka kode sumber.

**Hak Utama:**
- Digunakan untuk keperluan komersial dan non-komersial
- Dimodifikasi dan digabungkan dengan proyek lain
- Didistribusikan ulang dalam bentuk open-source maupun proprietary

**Kewajiban:**
- Mencantumkan copyright dan teks lisensi asli

**Catatan:**
- Tidak memberikan jaminan (no warranty)

---

## 2. BSD License (2-Clause / 3-Clause)

**Jenis:** Permissive License

**Deskripsi:**
Lisensi BSD mirip dengan MIT, namun versi 3-Clause menambahkan larangan
penggunaan nama pemegang hak cipta untuk keperluan promosi tanpa izin.

**Hak Utama:**
- Digunakan secara bebas, termasuk komersial
- Boleh dijadikan closed-source

**Kewajiban:**
- Mencantumkan lisensi dan hak cipta
- (3-Clause) Tidak menggunakan nama pencipta untuk promosi

---

## 3. Apache License 2.0

**Jenis:** Permissive License dengan perlindungan paten

**Deskripsi:**
Apache License 2.0 memberikan hak penggunaan luas serta perlindungan hukum
terkait klaim paten.

**Hak Utama:**
- Digunakan dan dimodifikasi secara komersial
- Digabungkan dengan software proprietary

**Kewajiban:**
- Mencantumkan lisensi dan perubahan yang dilakukan
- Menyertakan NOTICE file jika ada

**Catatan:**
- Cocok untuk proyek enterprise dan skala besar

---

## 4. GNU General Public License v3 (GPL v3)

**Jenis:** Strong Copyleft License

**Deskripsi:**
GPL v3 mewajibkan seluruh karya turunan untuk tetap menggunakan lisensi GPL
dan membuka kode sumbernya.

**Hak Utama:**
- Digunakan dan dimodifikasi secara bebas
- Didistribusikan ulang

**Kewajiban:**
- Kode sumber harus dibuka
- Lisensi turunan wajib GPL v3

**Catatan:**
- Tidak cocok untuk software proprietary

---

## 5. GNU Lesser General Public License (LGPL)

**Jenis:** Weak Copyleft License

**Deskripsi:**
LGPL dirancang untuk library agar dapat digunakan oleh software proprietary
tanpa kewajiban membuka seluruh kode.

**Hak Utama:**
- Digunakan dalam software closed-source
- Didistribusikan ulang

**Kewajiban:**
- Perubahan pada library LGPL harus dibuka

---

## 6. GNU Affero General Public License v3 (AGPL v3)

**Jenis:** Network Copyleft License

**Deskripsi:**
AGPL v3 memperluas GPL dengan kewajiban membuka kode sumber
meskipun software hanya digunakan sebagai layanan (SaaS).

**Hak Utama:**
- Digunakan dan dimodifikasi secara bebas

**Kewajiban:**
- Kode sumber wajib tersedia untuk pengguna layanan

**Catatan:**
- Cocok untuk mencegah eksploitasi open-source di cloud

---

## 7. Mozilla Public License 2.0 (MPL 2.0)

**Jenis:** Weak Copyleft (per file)

**Deskripsi:**
MPL mewajibkan pembukaan kode hanya pada file yang dimodifikasi,
sementara file lain dapat bersifat proprietary.

**Hak Utama:**
- Digunakan secara komersial
- Digabungkan dengan software tertutup

**Kewajiban:**
- File yang dimodifikasi harus tetap MPL

---

## 8. Proprietary / Closed Source License

**Jenis:** Komersial Tertutup

**Deskripsi:**
Lisensi proprietary membatasi akses, modifikasi, dan distribusi
sesuai perjanjian pemegang hak cipta.

**Hak Utama:**
- Digunakan sesuai kontrak atau perjanjian lisensi

**Kewajiban:**
- Tidak boleh mendistribusikan atau memodifikasi tanpa izin

---

## 9. Dual License

**Jenis:** Kombinasi Lisensi

**Deskripsi:**
Software disediakan dengan dua skema lisensi, biasanya
Open Source (misalnya GPL) dan Commercial License.

**Contoh Implementasi:**
- MySQL
- Qt Framework

---

## Ringkasan Lisensi

| Lisensi | Komersial | Wajib Open Source | Cocok untuk Enterprise |
|-------|-----------|------------------|----------------|
| MIT | Ya | Tidak | Ya |
| BSD | Ya | Tidak | Ya |
| Apache 2.0 | Ya | Tidak | Ya |
| GPL v3 | Ya | Ya | Tidak |
| LGPL | Ya | Sebagian | Ya |
| AGPL v3 | Ya | Ya (SaaS) | Tidak |
| MPL 2.0 | Ya | Sebagian | Ya |
| Proprietary | Ya | Tidak | Ya |

---

## Catatan Akhir

Pemilihan lisensi harus disesuaikan dengan:
- Model bisnis
- Kebutuhan komersial
- Pola distribusi (on-premise / SaaS)
- Risiko hukum dan paten



# Dataset License Reference (Kaggle)

Dokumen ini menjelaskan jenis-jenis lisensi dataset yang umum digunakan
pada platform Kaggle, beserta implikasi penggunaan, modifikasi,
dan distribusinya.

Dokumen ini bersifat informatif.
Untuk kepentingan hukum, selalu rujuk ke lisensi resmi yang tertera
pada halaman dataset Kaggle terkait.

---

## 1. CC0 – Public Domain

**Nama Lengkap:** Creative Commons Zero v1.0 Universal

**Deskripsi:**
Dataset dilepas ke domain publik tanpa batasan hak cipta.

**Hak Pengguna:**
- Digunakan untuk tujuan komersial dan non-komersial
- Dimodifikasi dan didistribusikan ulang
- Digabungkan dengan dataset lain

**Kewajiban:**
- Tidak ada kewajiban atribusi

**Catatan:**
- Lisensi paling bebas
- Sangat cocok untuk produk komersial dan riset terbuka

---

## 2. CC BY 4.0 – Attribution

**Nama Lengkap:** Creative Commons Attribution 4.0 International

**Deskripsi:**
Dataset dapat digunakan secara bebas dengan kewajiban atribusi
kepada pemilik dataset.

**Hak Pengguna:**
- Komersial dan non-komersial
- Modifikasi dan distribusi ulang

**Kewajiban:**
- Menyebutkan sumber dataset
- Menyertakan lisensi CC BY 4.0

---

## 3. CC BY-SA 4.0 – Attribution-ShareAlike

**Nama Lengkap:** Creative Commons Attribution-ShareAlike 4.0

**Deskripsi:**
Dataset bebas digunakan dan dimodifikasi, namun dataset turunan
wajib menggunakan lisensi yang sama.

**Hak Pengguna:**
- Digunakan secara bebas
- Dimodifikasi

**Kewajiban:**
- Atribusi
- Lisensi turunan harus CC BY-SA 4.0

**Catatan:**
- Kurang cocok untuk dataset turunan proprietary

---

## 4. CC BY-NC 4.0 – Attribution-NonCommercial

**Nama Lengkap:** Creative Commons Attribution-NonCommercial 4.0

**Deskripsi:**
Dataset hanya boleh digunakan untuk tujuan non-komersial.

**Hak Pengguna:**
- Riset
- Edukasi
- Proyek non-profit

**Larangan:**
- Penggunaan komersial
- Monetisasi langsung maupun tidak langsung

---

## 5. CC BY-NC-SA 4.0 – Attribution-NonCommercial-ShareAlike

**Nama Lengkap:** Creative Commons Attribution-NonCommercial-ShareAlike 4.0

**Deskripsi:**
Lisensi non-komersial dengan kewajiban ShareAlike
pada dataset turunan.

**Hak Pengguna:**
- Riset dan edukasi

**Kewajiban:**
- Atribusi
- Lisensi turunan harus sama

---

## 6. CC BY-ND 4.0 – Attribution-NoDerivatives

**Nama Lengkap:** Creative Commons Attribution-NoDerivatives 4.0

**Deskripsi:**
Dataset dapat digunakan dan didistribusikan ulang
namun tidak boleh dimodifikasi.

**Larangan:**
- Modifikasi data
- Preprocessing yang menghasilkan dataset turunan

**Catatan:**
- Tidak ideal untuk proyek machine learning

---

## 7. ODbL – Open Database License

**Nama Lengkap:** Open Data Commons Open Database License

**Deskripsi:**
Lisensi khusus untuk database terbuka dengan prinsip ShareAlike.

**Hak Pengguna:**
- Digunakan dan dimodifikasi
- Digabungkan dengan sistem lain

**Kewajiban:**
- Atribusi
- Database turunan wajib ODbL

---

## 8. Kaggle Dataset License (Custom)

**Nama Lengkap:** Kaggle Dataset Terms of Service / Custom License

**Deskripsi:**
Lisensi ditentukan langsung oleh pemilik dataset dan
tidak selalu mengikuti Creative Commons.

**Catatan Penting:**
- Wajib membaca deskripsi dataset
- Ketentuan dapat membatasi:
  - Penggunaan komersial
  - Redistribusi
  - Modifikasi

## Rekomendasi Penggunaan

- **Proyek Komersial / Startup**
  - CC0
  - CC BY 4.0

- **Riset Akademik**
  - CC BY 4.0
  - CC BY-NC 4.0

- **Machine Learning / Data Science**
  - Hindari CC BY-ND
  - Prioritaskan CC0 atau CC BY
