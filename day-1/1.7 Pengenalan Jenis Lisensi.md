# Software License Reference

Dokumen ini berisi ringkasan jenis-jenis lisensi perangkat lunak yang umum digunakan,
beserta karakteristik, hak, dan batasannya.

Dokumen ini bersifat informatif dan tidak menggantikan teks lisensi resmi.
Untuk penggunaan hukum, selalu rujuk ke dokumen lisensi asli.

---

## 1. MIT License

**Jenis:** Permissive License

**Deskripsi:**
MIT License mengizinkan penggunaan, modifikasi, distribusi, dan komersialisasi
perangkat lunak tanpa kewajiban membuka kode sumber.

**Hak Utama:**
- Digunakan untuk keperluan komersial dan non-komersial
- Dimodifikasi dan digabungkan dengan proyek lain
- Didistribusikan ulang dalam bentuk open-source maupun proprietary

**Kewajiban:**
- Mencantumkan copyright dan teks lisensi asli

**Catatan:**
- Tidak memberikan jaminan (no warranty)

---

## 2. BSD License (2-Clause / 3-Clause)

**Jenis:** Permissive License

**Deskripsi:**
Lisensi BSD mirip dengan MIT, namun versi 3-Clause menambahkan larangan
penggunaan nama pemegang hak cipta untuk keperluan promosi tanpa izin.

**Hak Utama:**
- Digunakan secara bebas, termasuk komersial
- Boleh dijadikan closed-source

**Kewajiban:**
- Mencantumkan lisensi dan hak cipta
- (3-Clause) Tidak menggunakan nama pencipta untuk promosi

---

## 3. Apache License 2.0

**Jenis:** Permissive License dengan perlindungan paten

**Deskripsi:**
Apache License 2.0 memberikan hak penggunaan luas serta perlindungan hukum
terkait klaim paten.

**Hak Utama:**
- Digunakan dan dimodifikasi secara komersial
- Digabungkan dengan software proprietary

**Kewajiban:**
- Mencantumkan lisensi dan perubahan yang dilakukan
- Menyertakan NOTICE file jika ada

**Catatan:**
- Cocok untuk proyek enterprise dan skala besar

---

## 4. GNU General Public License v3 (GPL v3)

**Jenis:** Strong Copyleft License

**Deskripsi:**
GPL v3 mewajibkan seluruh karya turunan untuk tetap menggunakan lisensi GPL
dan membuka kode sumbernya.

**Hak Utama:**
- Digunakan dan dimodifikasi secara bebas
- Didistribusikan ulang

**Kewajiban:**
- Kode sumber harus dibuka
- Lisensi turunan wajib GPL v3

**Catatan:**
- Tidak cocok untuk software proprietary

---

## 5. GNU Lesser General Public License (LGPL)

**Jenis:** Weak Copyleft License

**Deskripsi:**
LGPL dirancang untuk library agar dapat digunakan oleh software proprietary
tanpa kewajiban membuka seluruh kode.

**Hak Utama:**
- Digunakan dalam software closed-source
- Didistribusikan ulang

**Kewajiban:**
- Perubahan pada library LGPL harus dibuka

---

## 6. GNU Affero General Public License v3 (AGPL v3)

**Jenis:** Network Copyleft License

**Deskripsi:**
AGPL v3 memperluas GPL dengan kewajiban membuka kode sumber
meskipun software hanya digunakan sebagai layanan (SaaS).

**Hak Utama:**
- Digunakan dan dimodifikasi secara bebas

**Kewajiban:**
- Kode sumber wajib tersedia untuk pengguna layanan

**Catatan:**
- Cocok untuk mencegah eksploitasi open-source di cloud

---

## 7. Mozilla Public License 2.0 (MPL 2.0)

**Jenis:** Weak Copyleft (per file)

**Deskripsi:**
MPL mewajibkan pembukaan kode hanya pada file yang dimodifikasi,
sementara file lain dapat bersifat proprietary.

**Hak Utama:**
- Digunakan secara komersial
- Digabungkan dengan software tertutup

**Kewajiban:**
- File yang dimodifikasi harus tetap MPL

---

## 8. Proprietary / Closed Source License

**Jenis:** Komersial Tertutup

**Deskripsi:**
Lisensi proprietary membatasi akses, modifikasi, dan distribusi
sesuai perjanjian pemegang hak cipta.

**Hak Utama:**
- Digunakan sesuai kontrak atau perjanjian lisensi

**Kewajiban:**
- Tidak boleh mendistribusikan atau memodifikasi tanpa izin

---

## 9. Dual License

**Jenis:** Kombinasi Lisensi

**Deskripsi:**
Software disediakan dengan dua skema lisensi, biasanya
Open Source (misalnya GPL) dan Commercial License.

**Contoh Implementasi:**
- MySQL
- Qt Framework

## Ringkasan Lisensi

| Lisensi | Komersial | Wajib Open Source | Cocok untuk Enterprise |
|-------|-----------|------------------|----------------|
| MIT | Ya | Tidak | Ya |
| BSD | Ya | Tidak | Ya |
| Apache 2.0 | Ya | Tidak | Ya |
| GPL v3 | Ya | Ya | Tidak |
| LGPL | Ya | Sebagian | Ya |
| AGPL v3 | Ya | Ya (SaaS) | Tidak |
| MPL 2.0 | Ya | Sebagian | Ya |
| Proprietary | Ya | Tidak | Ya |

---

## Catatan Akhir

Pemilihan lisensi harus disesuaikan dengan:
- Model bisnis
- Kebutuhan komersial
- Pola distribusi (on-premise / SaaS)
- Risiko hukum dan paten
